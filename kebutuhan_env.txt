3. PROSEDUR INISIALISASI DATABASE (MENGGUNAKAN FILE BACKUP .TAR)
---------------------------------------------------------------
Karena database dideploy menggunakan file backup (.tar hasil pg_dump), IT Infra hanya perlu memulihkan/restore database tersebut ke server PostgreSQL baru.

Langkah Pemulihan Database:
1. Buat database baru di PostgreSQL (misal bernama `TA`).
2. Restore file database `.tar` menggunakan utilitas `pg_restore`:
   ```bash
   pg_restore -U [username] -d [nama_db_baru] -h [host_db] -p [port_db] [nama_file_backup].tar
   ```
   *Catatan: File tar ini sudah mencakup skema tabel, custom functions, triggers, stored procedures, serta semua master data (teknologi, role, dll.) sehingga tidak perlu menjalankan migrasi django (`migrate`) maupun script SQL secara manual.*
